# Phase 2: Machine Learning Baseline

## Objective

Predict whether the next day's stock price will increase or decrease.

---

## Target Variable Creation

Created binary classification target:

```text
1 = Price goes UP tomorrow
0 = Price goes DOWN tomorrow
```

Implemented using:

```python
df['Target'] =
(df['Close'].shift(-1) > df['Close'])
```

---

## Initial Feature Set

Used 11 features:

* Close
* Volume
* SMA_20
* SMA_50
* EMA_20
* EMA_50
* Daily_Return
* Volatility
* RSI
* MACD
* Signal_Line

---

## Data Splitting

Initially used:

```python
train_test_split()
```

Later discovered this was incorrect for stock prediction.

---

## Important Lesson Learned

Stock prediction is a:

```text
Time Series Forecasting Problem
```

and NOT a traditional classification problem.

Corrected using:

```python
Chronological Split
```

Example:

```text
Past Data -> Training
Future Data -> Testing
```

1. Stock prediction is a Time Series problem.
2. Chronological splitting is mandatory.
3. More features can reduce accuracy.
4. Feature engineering requires experimentation.
5. Random Forest can outperform XGBoost on noisy financial data.
6. Feature importance does not guarantee predictive power.
7. Machine Learning projects require iterative experimentation.

# Phase 3: Deep Learning (LSTM)

## Objective

Apply deep learning techniques to capture temporal patterns in stock market data.

---

## Why LSTM?

Traditional machine learning models treat each observation independently.

LSTM networks can learn sequential dependencies:

Day 1 → Day 2 → Day 3 → ... → Predict Tomorrow

---

## LSTM Architecture

Input:

* 10 previous trading days
* 11 features

Network:

* LSTM (64 units)
* Dropout (20%)
* LSTM (32 units)
* Dropout (20%)
* Dense (Sigmoid)

---

## Problems Faced

### Problem 1

Issue:
loss = NaN

Cause:
The dataset contained NaN values generated by:

* SMA indicators
* RSI
* Volatility
* Daily Returns

Solution:
Applied:

```python
df = df.dropna()
```

before scaling and sequence creation.
